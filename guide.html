// 01 SETUP

What you need

⬡ Raspberry Pi Zero 2 W (or Zero W)
▣ microSD card — 16 GB or larger
⚡ USB battery bank for power
⇄ USB OTG cable (micro-USB → USB-A female)
◈ USB-A to micro-USB cable (for FC)

Step 1 — Download the image

Grab the latest logfalcon-*.img.xz from GitHub Releases.

Step 2 — Flash the SD card

Use Raspberry Pi Imager or Balena Etcher. Any 16 GB+ card works. Write the .img.xz file — both tools decompress automatically.

Step 3 — Customise Wi-Fi (optional)

Before ejecting the SD card, open the boot partition and edit logfalcon-config.txt:

SSID=LogFalcon
PASSWORD=your-password
💡 TIP Default password is fpvpilot — change it before flying at a shared field. You can also change it later from the web UI → ⚙ Settings.

Step 4 — Boot

Insert the SD card into your Pi, power it with a battery bank. The LED will pulse slowly for ~60 seconds. When it goes solid on, the Pi is ready.

Alternative: Install on existing Pi

Already running Raspberry Pi OS? Install with one command:

curl -sSL https://github.com/proeugene/logfalcon/raw/main/scripts/install.sh | sudo bash

This downloads the correct ARM binary, installs packages, configures the hotspot, and enables all services. Reboot when done. To uninstall:

curl -sSL https://github.com/proeugene/logfalcon/raw/main/scripts/uninstall.sh | sudo bash

SSH access

SSH is enabled by default. Connect from any computer on the same network — or directly through the hotspot:

ssh pi@logfalcon.local   # or ssh pi@192.168.4.1 via hotspot
🔑 DEFAULT Username: pi  |  Password: logfalcon

Change your password immediately — especially before flying at a shared field:

passwd
💡 TIP The Wi-Fi hotspot password (fpvpilot) and SSH password are independent — changing one does not affect the other.

// 02 FIRST SYNC

①

Land your quad — FC flash is full or you want to clear it.

↓
②

Plug FC into the Pi — use the inner micro-USB port (OTG/data). The outer port is power only.

↓
③

Watch the LED — fast blink = syncing. Takes ~30–80 seconds depending on flash size.

↓
④

LED goes solid — done. Flash copied, verified (SHA-256), erased. Safe to unplug.

↓
⑤

Fly again. Repeat as many times as needed. Every sync is timestamped.

⚠ USB PORTS The Pi Zero has two micro-USB ports. Inner = data (OTG), outer = power. Plug your FC into the inner port. Plug your battery bank into the outer port.

// 03 LED REFERENCE

Only four patterns — designed to be unmistakable at a glance, even in direct sunlight.

●

Slow pulse (1s on / 1s off)

Pi is booting. Wait ~60 seconds.

●

Solid on

Ready. Pi booted, waiting for FC. Plug in now.

●

Steady fast blink

Sync in progress — copying, verifying, or erasing. Do not unplug.

●

Rapid burst → solid

Done. Safe to unplug. LED returns to solid ready state.

●

SOS pattern (repeating)

Error. Check the web UI or system logs for details.

// 04 DOWNLOADING LOGS

  1. Connect your phone or laptop to the LogFalcon Wi-Fi network.
  2. Captive portal opens automatically — like airport Wi-Fi. If it doesn't, type http://log.falcon in any browser.
  3. Browse sessions — grouped by FC (variant + UID), sorted by date.
  4. Download .bbl files → open in Blackbox Explorer.
📊 LIVE While the FC is syncing, the dashboard shows real-time progress — connect to the hotspot during a sync to watch it live. No need to wait for the LED; you can see exactly how much has copied, the current speed, and estimated time remaining. 
┌─────────────────────────────────────────────────┐
│  LogFalcon                    ⚙  [Syncing 45%] │
├─────────────────────────────────────────────────┤
│  Syncing flash… 45%  (2.1 / 4.0 MB)            │
│  █████████████░░░░░░░░░░░░░░░                   │
│                          1.2 MB/s · ~18s left   │
└─────────────────────────────────────────────────┘

The badge colour distinguishes each phase: Finding FC… (gray, pulsing) → Syncing N% (blue) → Verifying… (purple) → Erasing… (orange) → Idle (dim).

Storage layout on the Pi

/mnt/logfalcon-logs/
├── fc_BTFL_uid-12ab34cd/           ← Betaflight FC (by UID)
│   ├── 2026-02-26_143012/
│   │   ├── raw_flash.bbl           ← open in Blackbox Explorer
│   │   └── manifest.json           ← metadata + SHA-256
│   └── 2026-03-01_091000/
├── fc_INAV_uid-aabb1122/           ← iNav FC
│   └── 2026-03-02_101500/
└── fc_BTFL_uid-deadbeef/           ← another FC
    └── ...

Each session includes a manifest.json with FC info, file size, SHA-256 checksum, erase status, and timing data.

// 05 CONFIGURATION

Config lives at /etc/logfalcon/logfalcon.toml. Defaults work out of the box.

[general]
erase_after_sync = true               # false = copy only, don't erase
serial_baud = 921600

[hotspot]
hotspot_ssid = "LogFalcon"
hotspot_password = "fpvpilot"          # change this!

[storage]
storage_path = "/mnt/logfalcon-logs"
min_free_space_mb = 200                # always keep this much free
storage_pressure_cleanup = true        # auto-delete oldest when full

[web]
web_port = 80
idle_shutdown_minutes = 0              # 0 = never auto-shutdown
erase_after_sync Set false for dry-run mode — copies logs but never erases the FC.
min_free_space_mb Sync is refused if the Pi has less than this free. Prevents filling the SD card.
storage_pressure_cleanup When enabled, auto-deletes the oldest sessions to make room for a new sync.
idle_shutdown_minutes Auto-shutdown the web server after N minutes of inactivity. 0 = disabled.

// 06 FC COMPATIBILITY

Firmware Betaflight 4.0+ · iNav 2.6+
Blackbox device SPI Flash only
Flash chips W25Q128FV · W25Q64FV · M25P16
Protocol MSP v1 / v2 over USB CDC-ACM
Not supported FC-side SD card · Betaflight < 4.0 · ArduPilot
💡 HOW TO CHECK In Betaflight/iNav Configurator → Blackbox tab. If it shows FLASH with a size (16M, 64M, 128M), you're good. If it shows SD CARD or NONE, LogFalcon can't read it.

// 07 TROUBLESHOOTING

LED shows SOS / error pattern

Check the sync service log:

journalctl -u "logfalcon@ttyACM0" -n 50

Common causes: bad USB cable, unsupported FC variant, FC not in MSP mode.

FC not detected (no LED response after plug-in)
  • Make sure you're using the Pi's inner micro-USB port (OTG), not the power port.
  • Confirm your FC shows up as a serial port on a normal PC: /dev/ttyACM0 (STM32) or /dev/ttyUSB0 (CP2102/CH340).
  • Check the FC USB chip: lsusb — look for VID 0483 (STM32), 10c4 (CP2102), or 1a86 (CH340). All three are supported.
  • Try a shorter or better-quality USB cable.
Web UI not loading 
journalctl -u logfalcon-web -f

If the page won't load, try http://log.falcon or http://192.168.4.1 directly.

Sync seems slow

The Pi Zero W's single-core CPU is the bottleneck. Typical times:

1 MB~10–20s
2 MB~30–40s
4 MB~50–80s

Pi Zero 2 W is about 2× faster. Also try a shorter USB cable.

"FC uses SD card" error

Your FC logs to an SD card, not internal flash. MSP can't read FC-side SD cards.

In Configurator, set Blackbox Device = SPI Flash, or remove the FC's SD card and read it directly.

Can't SSH in / "connection refused"

SSH is enabled by default. Default credentials: user pi, password logfalcon.

ssh pi@logfalcon.local   # or ssh pi@192.168.4.1 via hotspot

If SSH is not responding, re-enable it from the Pi console:

sudo systemctl enable ssh --now
First boot shows "Please enter new username" on screen

This shouldn't happen with the LogFalcon image — the pi user is pre-configured and the first-boot wizard is disabled. If you see it, enter username pi and password logfalcon. Then verify you used the correct logfalcon-*.img.xz image from GitHub Releases.

"Flash is empty" message

The FC flash has no data to sync. This is normal if you already synced or never flew with logging enabled.

Verify in Configurator that Blackbox Logging is enabled and fly a pack first.

// 08 BUILDING FROM SOURCE

LogFalcon is written in Go. Requires Go 1.23+.

Clone and build

git clone https://github.com/proeugene/logfalcon
cd logfalcon
make build          # native binary
make build-pi       # cross-compile for Pi Zero W (ARM6)
make build-pi2      # cross-compile for Pi Zero 2 W (ARM64)

Run tests

make test            # runs go test -race ./...
make lint            # runs golangci-lint

All tests run without hardware — serial ports, GPIO, and filesystem are mocked via interfaces.

Install on a Pi

sudo bash scripts/install.sh

Or to install a freshly built binary on a Pi you can already reach via SSH:

scp bin/logfalcon-arm6 pi@logfalcon.local:/tmp/
ssh pi@logfalcon.local 'sudo install -m 755 /tmp/logfalcon-arm6 /opt/logfalcon/logfalcon && sudo systemctl restart logfalcon-web'

Project structure

cmd/logfalcon/       CLI entry point
internal/
├── config/          TOML config loader
├── msp/             MSP protocol: framing, CRC, Huffman, client
├── fc/              FC detection and handshake
├── sync/            10-step sync orchestrator
├── storage/         Session dirs, manifest, stream writer
├── web/             HTTP server, SSE, captive portal, downloads
├── led/             LED state machine (sysfs + GPIO)
└── util/            Disk space utilities

// 09 CLI REFERENCE

logfalcon                              # sync (auto-detect port)
logfalcon --port /dev/ttyACM0          # specific serial port
logfalcon --dry-run                    # copy only, don't erase
logfalcon --web                        # start web server only
logfalcon --web --port 8080            # web server on custom port
logfalcon --config /path/to/file.toml  # custom config file
logfalcon --version                    # print version
--web Start the web UI server instead of running a sync.
--port Serial port path (sync mode) or listen port number (web mode).
--dry-run Copy the flash but skip erase. Useful for testing.
--config Path to TOML config file. Default search: /etc/logfalcon/ → ./config/
--version Print version and exit.
